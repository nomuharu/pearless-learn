## 背景（実験で確定した事実）

1. **方向は予測できない**: 5分先の UP/DOWN は OHLCV 由来16特徴量では正答率 50〜54%
   が上限（方向専用 lstm_dir でも 50.3%）。
2. **動くことは予測できる**: lstm_focal の p_move = p_up + p_down は基底率の
   3.2 倍のリフト。これが唯一の確実なエッジ。
3. **OCOブレイクアウト戦略は期待値プラス**: val 選択の運用点（t_move=0.94,
   δ=2.5銭）を test に適用して +2.6〜3.1銭/トレード（スリッページ 0.3〜0.5銭込み）。
4. **損失の主因は往復バー**: 同一 M1 内で上下両トリガーに到達するトレードの期待値は
   厳密に −δ−spread。選択運用点では全トレードの約 45% を占める。
5. **エッジは高ボラ期に集中**: 利益の大半が 2024-04〜09（円急変動期）。
   直近12ヶ月（2025-05〜2026-04）は合計マイナス。

ベースライン（改善の比較基準）:
- test 期間 2024-03〜2026-04、481 トレード、平均 +3.10銭/トレード（slip 0.3銭）、
  クリーンブレイク勝率 66.2%、最悪シード最大DD 298銭/標準ロット

## ゴール

OCO 戦略の期待値と直近レジームでの安定性を改善する。優先順:

- **Phase B（決済ルール改善）**: 平均損益/トレードをベースライン比で改善する
  決済ルールを特定する（学習不要、バックテストのみ）
- **Phase A（往復バー予測）**: 「クリーンブレイク vs 往復」分類モデルで
  往復トレードを事前に弾き、期待値を改善する
- **Phase E（walk-forward 再学習）**: 月次ローリング再学習で直近レジームの
  劣化を抑える

## Phase B: 決済ルール改善（学習不要・即日）

現状の「次の5分足の終値で決済」を以下のバリアントと比較する。
すべて M1 パス再構成バックテスト（scripts/backtest_oco.py 拡張）で検証。

### Tasks

- [x] B-1: バックテストに決済ルールのバリアントを実装（scripts/backtest_oco_exits.py）
- [x] B-2: val でルール+パラメータを選択 → test に1点適用（リークなし手順）
- [x] B-3: 月次内訳（直近12ヶ月の成績）とDDをベースラインと比較し採否判定

### 結果（2026-06-11 完了・全バリアント棄却）

- ストップ系（TP/SL・トレーリング・建値）は **val 全設定で期待値マイナス**
  （−0.6〜−2.1銭）。高ボラ局面のバー内ノイズでストップが刈られるため
- hold6 は val +8.6銭 → **test −0.05銭・最悪DD 701銭**。val 期間への過適合で不採用
- **結論: ベースライン（次の5分足終値決済）を維持**。エッジは次の1本に集中して
  おり、保有を延ばしても増えない（horizon=1 のラベル設計と整合）

## Phase A: クリーンブレイク vs 往復の分類モデル

### 設計

- ラベル: moveシグナル発火バーについて、M1 パスで「片側トリガーのみ到達=クリーン(1)
  / 両側到達=往復(0) / 不到達=対象外」を機械生成（バックテストロジックを流用）
- 学習データ: p_move 上位のバーに限定せず、全バーのうち「±δ にいずれか到達した」
  バーを対象とする（シグナル外も含めることでサンプル数を確保）
- モデル: LSTMModel ベース（MODEL_CONFIGS に "lstm_clean" を追加）。
  入力は既存16特徴量ウィンドウ（npy 共通、列選択なし）
- 推論時の合議: p_move >= t_move AND p_clean >= t_clean で OCO 設置

### Tasks

- [x] A-1: ラベル生成スクリプト（scripts/make_clean_labels.py）。
      pearless-usdjpy-m5 ではなく軽量な pearless-aux-labels dataset に分離
- [x] A-2: MODEL_CONFIGS へ "lstm_clean" 追加 + train_lstm_clean.ipynb 生成
- [x] A-3: Kaggle で学習（16エポック、val: 往復F1 0.43 / クリーンF1 0.77）
- [x] A-4: バックテストに p_clean ゲートを統合し比較（scripts/backtest_oco_clean.py）

### 結果（2026-06-11 完了・棄却）

- 全バー上では往復の判別力あり（往復F1 0.43）だが、**p_move>=0.94 の
  シグナルバー条件下では判別力がほぼ消失**（p_clean 分位別の実往復率
  71.9/72.5/71.7/63.3% と最上位でも 9pt 差）
- val 選択ゲートの test 平均 2.54銭 vs ベースライン 2.80銭で改善なし
- 解釈: lstm_clean が学習したのは「高ボラ→往復しやすい」で p_move と同じ情報。
  シグナル条件下の「伸びる/暴れる」を分ける直交情報は 16特徴量には無い
  （方向予測の失敗と同根。負の結果として記録）

## Phase E: walk-forward 再学習

### 設計

- 月次ローリング: 各月初に「その時点までのデータ」で lstm_focal（+ 採用されたら
  lstm_clean）を再学習し、翌1ヶ月を推論する
- 計算量を抑えるため test 期間の 26 ヶ月のうち、四半期ごと（約9回）の再学習で近似
- 運用点（t_move 等）は各再学習時点の直近 val 窓で選び直す

### Tasks

- [x] E-1〜E-2: 単一 Kaggle カーネル内で test 期間を9チャンクに分割し、
      ウォームスタート連鎖の逐次 fine-tune を実行（notebooks/train_walkforward.ipynb。
      期間別 npy 再生成・dataset 再アップロードは不要な設計に変更）
- [x] E-3: walk-forward vs 固定モデルの比較（チャンク別に直前 val tail の分位で
      閾値を選び直し、シグナル頻度を固定モデルに揃えて比較）

### 結果（2026-06-11 完了・採用）

| | 固定モデル | walk-forward |
|---|---|---|
| トレード数（test 2.2年） | 481 | 818 |
| 平均損益/トレード | +2.80銭 | +1.25銭 |
| 期間合計 | 13.5円/ドル | 10.2円/ドル |
| **直近12ヶ月（2025-05〜）** | **−81,300円/Lot** | **+48,400円/Lot** |

- **完了条件を達成**: 直近12ヶ月がマイナスからプラスに反転。月次の偏りも大幅に
  平準化（固定モデルは2024年に利益集中、walk-forward はほぼ毎月シグナルが出る）
- トレードあたりのエッジは薄まる（閾値の都度選択で頻度が増えるため）が、
  時間的安定性が大きく改善。**運用形態として walk-forward を採用**
- 残課題: 2026-04 はどちらも負け（−11万円/Lot）。直近レジームの完全な追従は不可。
  チャンク別の実シグナル数のばらつき（1〜418件）も閾値転送の不安定さを示す

## Verification Strategy

- すべての比較は「val でルール/閾値/モデルを選び、test には選んだ1点のみ適用」の
  リークなし手順で行う
- 往復バーの解決は理論期待値（−δ−spread）算入を基本とし、DD 計算時のみ
  50/50 ランダム解決の複数シードで評価
- 月次内訳（特に直近12ヶ月）を必ず併記し、平均値のレジーム偏りを監視する

## リスク

- Phase A のラベル不均衡（クリーン:往復 比は δ に依存）→ δ 固定でラベル生成し、
  クラス重みは既存の training 基盤で対応
- Kaggle GPU 週次クォータ（30h）→ Phase E は四半期粒度で近似し学習回数を 9 回に抑制
- 2026-04 のような直近の負け月は walk-forward でも救えない可能性（レジーム依存の
  本質的限界）→ その場合は「実現ボラによる追加ゲート」を Phase E 後の検討事項とする
